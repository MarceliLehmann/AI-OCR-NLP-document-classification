# Security and Confidentiality

This repository is intentionally sanitized.

The original project was created in a commercial environment, so only safe, high-level information is included here.

---

## Why Sanitization Matters

AI document processing systems often work with sensitive content.

Even if source code seems harmless, it may reveal:

- client names,
- internal architecture,
- private endpoints,
- business logic,
- document structures,
- security assumptions,
- deployment details,
- private package sources.

For that reason, the public version must avoid exposing original implementation details.

---

## Removed from Public Version

The public portfolio version does not include:

- production source code,
- real input documents,
- real output files,
- internal logs,
- private IP addresses,
- private registry URLs,
- Docker registry details,
- CI/CD credentials,
- client names,
- confidential configuration,
- generated embeddings from real data.

---

## Safe to Show Publicly

The following information is generally safer for a portfolio repository:

- high-level architecture,
- technology categories,
- sanitized diagrams,
- synthetic examples,
- pseudocode,
- personal contribution summary,
- lessons learned,
- general engineering decisions.

---

## Public Naming Recommendation

Use neutral names such as:

- AI Document Routing Pipeline
- OCR Metadata Extraction Case Study
- Public-Sector Document Processing Case Study

Avoid using the real client name or internal project name unless you have written permission.
